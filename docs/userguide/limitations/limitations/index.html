制限事項

次の項では、WebLogic Deploy Toolingの既知の制限について説明します。 各問題には、回避策または関連する問題番号が含まれている場合があります。

WebLogic Kubernetes Operator WebLogic Deploy Tooling 4.0.0 (およびそれ以降)を使用すると、イメージ内のモデル・ドメインが失敗

問題: Model-in-Image (MII)を使用してWebLogic Kubernetes Operatorドメインをデプロイすると、イントロスペクタの失敗が発生します。

処置: WDT 4.0は、アーカイブ・ファイル・パス処理に大幅な変更を加え、WebLogic Kubernetes Operator 4.2.0から使用可能になりました。 問題を解決するには、WebLogic Kubernetes Operator 4.2.0+にアップグレードします。

Discover Domain ToolのSEVEREメッセージ

問題: discoverDomain STDOUTには、12.2.1.0ドメインに対して実行する場合、cd()およびls()に関する多数のSEVEREメッセージが含まれます。 Discover Domain Toolは、ドメインMBeansをWLSTを使用してナビゲートし、ドメインに存在するMBeansを決定します。 存在しないMBeanをテストすると、WLSTによってエラー・メッセージが記録されます。 このWLSTの問題に対処するために使用できる12.2.1.0 PSUはありません。 12.2.1.1で解決されます。

処置: 12.2.1.0ドメインの検出中に記録された次のメッセージは無視します。

<Jan 14, 2019 1:14:21 PM> <SEVERE> <CommandExceptionHandler> <handleException> <> <Error: cd() failed.>
<Jan 14, 2019 1:14:21 PM> <SEVERE> <CommandExceptionHandler> <handleException> <> <Error: ls() failed.>

NMProperties KeyStores属性を持つCreate Domain Tool

問題: Create Domain Toolでは、モデルのtopology:/NMProperties/KeyStoresが正しく設定されないため、ドメインはnodemanager.propertiesのKeyStoresエントリなしで作成されます。

処置: これは、WLSTバグ37587121によるものです。 Oracle Supportに連絡して、環境に適用する適切なパッチを取得します。

11g JRFドメインを含むCreate Domain Tool

問題: Create Domain Toolは、11g JRFドメインのRCUを初期化できません。 ツールはログにエラー・メッセージを発行し、作成プロセスを終了します。

処置: JRFドメインに対してcreateDomainを実行する前に、WLS rcuコマンドを実行

セキュリティ構成の資格証明

問題: 14.1.1より前のWLSバージョンの場合、topology/SecurityConfigurationフォルダにCredentialEncrypted属性の設定に問題があります。 値は構成で適切に暗号化されず、ドメインはエラーで起動に失敗します: 

java.lang.IllegalArgumentException: In production mode, it's not allowed to set a clear text value to the property: CredentialEncrypted of SecurityConfigurationMBean

処置: Oracle Supportに連絡して、ツールを実行する前に、WebLogic Serverバージョンのバグ番号30874677のパッチを取得してください。

RotateLogOnStartup属性の設定に関する問題

問題: 既存のWLSバージョンの場合、様々なログ・ファイル・フォルダにRotateLogOnStartup属性の設定に問題があります。 値は正しく保持されず、ドメインの起動時に割り当てが存在しません。

処置: Oracle Supportに連絡して、ツールを実行する前に、WebLogic Serverバージョンのバグ番号29547985のパッチを取得してください。

Discover Domainツールでユーザーまたはグループが検出されない

問題: ドメインの検出では、構成済の認証プロバイダ・タイプで定義されたユーザーおよびグループの検出は試行されません。

処置: これは、ユーザーおよびグループ・ストアとしてWebLogic Server内で実行される組込みLDAPサーバーを使用するDefaultAuthenticatorを使用するドメインにのみ問題となる必要があります。 DefaultAuthenticator内のユーザーおよびグループを検出するには、DefaultAuthenticatorを含む引数(ALLやDefaultAuthenticator,DefaultCredentialMapperなどのプロバイダ・タイプのカンマ区切りリストなど)を指定して-discover_security_provider_dataスイッチを使用します。

クリア・テキスト・パスワードを含むJRFドメイン構成ファイル

問題: JRFドメインが作成されると、jps-config.xmlおよびjps-config-jse.xmlファイルには、キー・ストアのクリア・テキスト・パスワードが含まれます。

処置: Oracle Walletを使用してキー・ストアを作成し、JVMのキー・ストア・プロバイダの優先度を変更する必要があります。 詳細は、Oracle Support Doc ID 2215283.1を参照してください。 Oracle Autonomous Transaction Databaseを使用してJRFドメインを作成する場合、JKSキー・ストアのかわりにSSOキー・ストアを使用でき、生成されたjps-config.xmlおよびjps-config-jse.xmlファイルは、クリア・テキスト・パスワードなしでSSOキー・ストアを使用します。 

domainInfo:
    RCUDbInfo:
     databaseType : ATP
     rcu_prefix : FMW
     rcu_schema_password : '...'
     rcu_admin_password: '...'
     rcu_db_user : admin
     tns.alias : myatp_tp
     javax.net.ssl.keyStoreType: SSO
     javax.net.ssl.trustStoreType: SSO

SSH使用

問題: WDT Create DomainはSSHをサポートしていないようです。

処置: WDT Create DomainではSSHがサポートされていないため、ドメインの作成は、生成されたドメイン・ファイルのパスが実際のJavaおよびOracle Homesを指していることを確認するために、ドメインが存在するマシンで実行する必要があります。 SSHを介したCreate Domainをサポートするには、WDTがリモート・マシンにインストールしてから、そこでコマンドを実行する必要があります。 回避方法は、ユーザーが一時的にWDTをインストールし、ドメインを作成してWDTをアンインストールすることです。

問題: WDTは、@@JAVA_HOME@@、@@PWD@@および@@TMPパス・トークンを正しく処理しません。

処置: WDTは現在、ユーザーがこれらのパスに「リモート」値を指定する方法を提供しないため、SSHコンテキストでこれらのパス・トークンを使用しないでください。

バージョン設定されたアプリケーションの本番再デプロイメント

問題: WDT Update Domainは、中断のない更新をサポートするために、アプリケーションの新しいバージョンを正しくデプロイしません。

処置: 今後のリリースでこの問題を解決する予定ですが、WebLogic Serverドキュメントで説明されているように、weblogic.Deployerツールを使用して、新しいソース・ファイルを指すアプリケーションを再デプロイします。

JDK 8で実行されているオンラインUpdate Domain

問題: 同じUpdate DomainコールでRemoteAnonymousRmiiiopEnabledおよびRemoteAnonymousRmit3Enabledの値を更新しようとすると、WDT Update Domainが失敗することがあります。

####<Jan 6, 2025 12:38:06 PM> <SEVERE> <update> <main> <WLSDPLY-09015> <updateDomain deployment failed: activate() failed: weblogic.rjvm.PeerGoneException: ; nested exception is: 
	java.io.EOFException>

        1. WLSDPLY-09012: While handling an error, failed to stop the current edit session and disconnect: undo() failed: None

SEVERE Messages:

updateDomain.sh failed (exit code = 2)

処置: これは、WebLogic Server Bug 37443991によって発生します。 この問題を解決するには、Oracle SupportでSRを申請し、バグ37443991のパッチを依頼してください。 既知の回避策は、これら2つの更新を別々のオンライン更新ドメイン呼び出しに分割するか、オフラインUpdate Domainを使用することです。
