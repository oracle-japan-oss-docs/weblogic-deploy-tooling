Create Domain Tool

Create Domain Toolでは、モデルとWLSTをオフラインで使用してドメインを作成します。 ツールを使用するには、次に示すように、モデルのdomainInfoセクションにドメインの管理パスワードを指定する必要があります。 

domainInfo:
    AdminPassword: welcome1

前述のモデルを使用して、createDomainツールを実行し、作成するドメインのタイプおよび作成場所を指定します。

$ weblogic-deploy\bin\createDomain.cmd -oracle_home c:\wls12213 -domain_type WLS -domain_parent d:\demo\domains -model_file MinimalDemoDomain.yaml

明らかに、テンプレート定義のサーバーのみを使用して空のドメインを作成することはあまり面白くありませんが、この例では、モデルのスパースがより強まるようになります。 Create Domain Toolを実行する場合、モデルはアーカイブ・ファイル内またはスタンドアロン・ファイルとして提供する必要があります。 アーカイブ・ファイルとモデル・ファイルの両方が指定されている場合、アーカイブ外のモデル・ファイルは、アーカイブ内のモデル・ファイルよりも優先されます。 アーカイブ・ファイルが指定されていない場合、Create Domain Toolでは、ドメイン内のモデルのtopologyセクションのみ(domainInfoセクションを使用)が作成されます。 これは、モデルのresourcesおよびappDeploymentsセクションがアーカイブからファイルを参照できるため、モデル定義のリソースおよびアプリケーションでドメインを作成するために、アーカイブ・ファイルを指定する必要があります(モデルがアーカイブ内の何も参照していない場合でも)。 将来のある時点で、この制限は、実際に必要とされている場合にのみアーカイブを必要とするように緩和できます。

Create Domain Toolでは、WLS、RestrictedJRF、およびJRFの3つのドメイン・タイプが認識されます。 ドメイン・タイプを指定する場合、Oracle Homeはドメイン・タイプの要件と一致する必要があります。 RestrictedJRFとJRFの両方に、FMWインフラストラクチャ(JRFとも呼ばれる)がインストールされているOracle Homeが必要です。 JRFドメインを作成する場合、RCUデータベース情報をcreateDomainスクリプトの引数として指定する必要があります。 必要なパスワードの入力を求められることに注意してください。 オプションで、スクリプトの標準入力(stdinなど)にパイプして、ユーザー入力なしでスクリプトを実行することもできます。 たとえば、JRFドメインを作成するコマンドは次のようになります。 このコマンドを実行する前に、ユーザーがRCUを実行する必要があることに注意してください。 

$ weblogic-deploy\bin\createDomain.cmd -oracle_home c:\jrf12213 -domain_type JRF -domain_parent d:\demo\domains -model_file DemoDomain.yaml -rcu_db mydb.example.com:1539/PDBORCL -rcu_prefix DEMO [-rcu_db_user SYS]

Create Domain ToolでRCUを実行するには、前のコマンドラインに-run_rcu引数を追加し、RCUスキーマが自動的に作成されます。 ツールでRCUを実行すると、新しいスキーマを作成する前に、同じRCUプレフィクスですでに存在する競合するスキーマが自動的に削除されることに注意してください。

また、コマンドライン引数を使用するかわりに、モデル内の接続情報を指定することもできます。 これは、複雑なデータベース接続文字列と、RACやOracle Autonomous Transaction Processing Cloud Serviceデータベースなどの追加パラメータを必要とするデータベースで特に簡単です。 使用方法については、「モデルでのRCU接続情報の指定」を参照してください。

より複雑なドメインを作成するには、カスタム・ドメイン・タイプを作成する必要がある場合があります。 これは、ドメインにカスタム・テンプレートまたは他のOracle製品のテンプレートがある場合に便利です。 詳細については、「ドメイン・タイプ定義」を参照してください。

暗号化モデルの使用

モデルまたは変数ファイルにWDT暗号化ツールで暗号化されたパスワードが含まれている場合は、コマンド行で-use_encryptionフラグを使用して作成中にパスワードを復号化して、暗号化が使用されていることをCreate Domain Toolに指示し、暗号化パスフレーズを要求します。 データベース・パスワードと同様に、ツールは標準入力(stdinなど)からパスフレーズを読み取って、ユーザー入力なしでツールを実行できます。 標準入力プロンプトは、他の2つのオプションでバイパスできます。 他の2つのオプションを使用して、stdinプロンプトをバイパスできます: パスフレーズを環境変数に格納し、コマンド行オプション-passphrase_envで環境変数名を使用するか、パスフレーズの単一値でファイルを作成します。 コマンド行オプション-passphrase_fileを使用してファイルの名前を指定します。 パスフレーズは、ツールによってファイルから読み取られます。

複数のモデルの使用

Create Domain Toolでは、「複数のモデルの使用」に説明されている複数のモデルを使用できます。

開発ドメインおよびboot.properties

開発ドメインを作成する場合、WDTは、ドメイン内の各サーバーに対してboot.propertiesファイルを作成する利便性を提供します。 boot.propertiesファイルには、管理サーバー・ユーザー名およびパスワードの暗号化された値が含まれます。 管理サーバーまたは管理対象サーバーを起動すると、WebLogic Serverは資格証明のプロンプトを省略し、かわりにboot.propertiesファイルの資格証明を使用します。

ServerStartModeオプションがprodに設定されている場合、またはドメインProductionModeEnabledがtrueに設定されている場合、ドメインは本番モードになります。 これらの両方の属性のデフォルト値は開発モードです。

boot.propertiesファイルは、WDTが実行されるマシンのドメイン・ホームに格納されます。 サーバーごとに<domain_home>/servers/<server_name>/security/boot.propertiesとして格納されます。

次に、両方の属性を明示的に開発モードに設定したモデルの例を示します。

domainInfo:
    AdminUserName: weblogic
    AdminPassword: welcome1
    ServerStartMode: dev
topology:
    Name: my-domain
    AdminServerName: admin-server
    ProductionModeEnabled: false

createDomainのパラメータ表

パラメータ 定義 デフォルト
-archive_file 使用するアーカイブ・ファイルへのパス。 -model_file引数が指定されていない場合、このアーカイブのモデル・ファイルが使用されます。 これは、アーカイブ・ファイルのカンマ区切りリストとして指定することもできます。 各アーカイブ内の内容が、リストの前のアーカイブより優先されます。
-domain_home -domain_parentを使用しない場合は必須です。 ドメインを作成する完全なディレクトリと名前。
-domain_parent -domain_homeを使用しない場合は必須です。 ドメインを作成する親ディレクトリ。 名前は、モデル内のドメイン名です。
-domain_type ドメインのタイプ(WLS、JRFなど)。 WLS
-java_home 新しいドメインに使用するJavaホーム。 指定しない場合、JAVA_HOME環境変数の値がデフォルトになります。
-model_file モデル・ファイルのロケーション。 これは、モデル・ロケーションのコンマ区切りリストとして指定することもできます。このリストでは、前のモデル・ロケーションの上に続く各モデル層が配置されます。
-oracle_home Oracle WebLogicインストールのホーム・ディレクトリ。 ORACLE_HOME環境変数が設定されていない場合は必須です。
-opss_wallet_passphrase_env プロンプトでOPSSウォレット・パスフレーズを入力する代替手段。 この値は、WDTがパスフレーズの取得に使用する環境変数名です。
-opss_wallet_passphrase_file プロンプトでOPSSウォレット・パスフレーズを入力する代替手段。 値は、パスフレーズを取得するためにWDTが読み取る文字列値を持つファイルの名前です。
-passphrase_env プロンプトで暗号化パスフレーズを入力する代替手段。 この値は、WDTがパスフレーズの取得に使用する環境変数名です。
-passphrase_file プロンプトで暗号化パスフレーズを入力する代替手段。 値は、パスフレーズを取得するためにWDTが読み取る文字列値を持つファイルの名前です。
-rcu_database RCUデータベース接続文字列。
-rcu_prefix 使用するRCUプレフィクス。
-rcu_db_user 使用するRCU dbUser。 sys
-run_rcu RCUを実行して、指定したRCUプレフィクスを使用してドメイン・タイプで指定されたデータベース・スキーマを作成します。 RCUを実行すると、作成を試みる前に、同じRCUプレフィクスを持つ既存のスキーマが削除されます。
-use_encryption モデルまたは変数ファイル内の1つ以上のパスワードが暗号化され、復号化される必要があります。 この機能にはJava 8以降が必要です。
-variable_file モデルで使用される変数の値を含むプロパティ・ファイルのロケーション。 これは、プロパティ・ファイルのカンマ区切りリストとして指定することもできます。このリストでは、前述のプロパティ・レイヤーの先頭に続く各プロパティ・レイヤーのセットがあります。
