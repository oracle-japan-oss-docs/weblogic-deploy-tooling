Discover Domain Tool

Discover Domain Toolは、既存のドメインを検査し、そこから構成およびバイナリを収集することでモデルおよびアーカイブ・ファイルを作成するためのブートストラップ・メカニズムを提供します。 Discover Domain Toolでは既存のドメインからパスワードが検出されないため、ツールによって生成されたモデル・ファイルはCreate Domain ToolまたはDeploy Applications Toolによって直接使用できないことに注意してください。 かわりに、検出されたパスワードの--FIX ME--プレースホルダーを配置します。 ドメイン・ユーザーも検出できないため、ツールはdomainInfoセクションのAdminUserNameおよびAdminPasswordフィールドに同じプレースホルダー値をインジェクトします。 このツールの考え方は、生成されたモデルおよびアーカイブ・ファイルを、他のツールの1つを実行するためのニーズに合わせて編集できる開始点を提供するだけです。

Discover Domain Toolを実行するには、Oracleホームおよびドメインのロケーションを指定するだけです。 検出されたドメイン情報が書き込まれるモデル・ファイルおよびアーカイブ・ファイルのロケーションとファイル名を指定します。 モデルは、YAML形式またはJSON形式のいずれかで生成できます。 正しいサフィクスを使用してファイル名にラベルを付けます。 shスクリプトとcmdスクリプトの両方が提供されています。

基本的なdiscoverDomainの例:

$ weblogic-deploy\bin\discoverDomain.cmd -oracle_home c:\wls12213 -domain_home domains\DemoDomain -archive_file DiscoveredDemoDomain.zip -model_file DiscoveredDemoDomain.yaml

アーカイブを作成するときに、次の注意事項とともにドメイン構成によって参照されるすべてのバイナリ、スクリプト、および必要なディレクトリを収集しようとします。

  1. ORACLE_HOMEから参照されるバイナリは、モデルドリブン操作が適用されるターゲット・ドメインに存在想定されるため、収集されません。 これは、モデルがWebLogic Serverバージョンに依存しないようにするための鍵です。
  2. 現在のフォームでは、Discover Domain Toolはローカル・マシンからアクセス可能なバイナリおよびスクリプトのみを収集します。 警告は、見つからないバイナリまたはスクリプトに対して生成されますが、可能な場合はそれらのバイナリの構成が引き続き収集されます。 欠落しているファイルを適切なロケーションのアーカイブに追加し、必要に応じてモデルを編集して、アーカイブ内の相対パス(たとえば、wlsdeploy/applications/myapp.ear)を使用してアーカイブ内のファイルを指すようにする必要があります。
  3. モデル・ファイルを検査する場合は、アーカイブ・ファイルを生成せずにDiscover Domain Toolを実行できます。 作成または更新ドメインでは、ドメインにインストールされるバイナリ、スクリプト、またはディレクトリに対して有効なアーカイブ・ファイルが必要です。

可変ファイルのロケーションと名前を指定することで、モデルで生成されるパスワード属性をカスタマイズできます。 このファイルは、モデルで見つかった各パスワードのkey=valueを含むテキスト・プロパティ・ファイルです。 キーはパスワード属性の一意のトークン名であり、値は置換値であり、この場合空の文字列です。 モデル内の属性は、トークン名とプロパティ・フィールド表記でインジェクトされます。 たとえば、@@PROP:AdminUserName@@または@@PROP:JDBCSystemResource.<Name>.JdbcResource.JDBCDriverParams.PasswordEncrypted@@です。

変数ファイル名を含むコマンド行の例:

$ weblogic-deploy\bin\discoverDomain.cmd -oracle_home c:\wls12213 -domain_home domains\DemoDomain -archive_file DiscoveredDemoDomain.zip -model_file DiscoveredDemoDomain.json -variable_file DiscoverDemoDomain.properties

オンライン・モードは、管理サーバーまたはリモート・マシンで使用できます。 管理サーバーから実行している場合は、ドメイン・ホーム・ディレクトリ-domain_homeのロケーションを指定できます。

リモート・マシンから実行している場合は、ドメイン・ホーム・ディレクトリの-domain_homeオプションを指定する必要はありません

オンラインWLSTを使用してドメインを検出するには、コマンドラインに管理ユーザー名および管理URLを含めます。 ツールにより、STDINにパスワードを入力するよう求められます。 プロンプトを省略するには、2つのオプションのいずれかを使用できます。 パスワードを環境変数に格納し、コマンド行オプション-admin_pass_envで変数名を使用します。 パスワード値をファイルに格納します。 コマンドライン・オプション-admin_pass_fileを使用してファイル名を指定します。

管理サーバーからオンラインWLSTモードで実行する例:

$ weblogic-deploy/bin/discoverDomain.sh -oracle_home /u01/oracle -domain_home /u01/oracle/domains/DemoDomain -archive_file ./DiscoveredDemoDomain.zip -model_file ./DiscoveredDemoDomain.yaml -admin_user weblogic -admin_url t3://localhost:7001

リモート・マシンからオンラインWLSTモードで実行する例:

$ weblogic-deploy/bin/discoverDomain.sh -oracle_home /u01/oracle -remote -model_file ./DiscoveredDemoDomain.yaml -admin_user weblogic -admin_url t3://localhost:7001

対応するバイナリをアーカイブ・ファイルに正常に収集するには、ドメイン・バイナリがあるのと同じシステム上でコマンドを実行する必要があります。

カスタム・テンプレートまたは製品テンプレートを使用してドメインが作成されると、テンプレートによって、モデルで検出したりアーカイブに収集する必要がないドメインにリソースがインストールされます。 ドメイン・タイプ引数(ドメイン・タイプ指定ファイルに対応)は、これらのリソースおよびファイルを無視するために、ドメインのタイプを記述する必要があります。 デフォルトでは、ドメイン・タイプWLSを使用して検出が実行され、WebLogic Serverテンプレートのみがドメインに適用されたとみなされます。 ツールには、他の2つのドメイン・タイプ(RestrictedJRFおよびJRF)のタイプのタイプが定義されています。 これらのドメイン・タイプ、または別のカスタム・タイプ・リーフを使用できます。 詳細については、「ドメイン・タイプ定義」を参照してください。

ドメイン・タイプ引数を使用する例:

$ weblogic-deploy\bin\discoverDomain.cmd -oracle_home c:\wls12213 -domain_home domains\DemoDomain -archive_file DiscoveredDemoDomain.zip -model_file DiscoveredDemoDomain.yaml -domain_type RestrictedJRF

モデルがモデル・ファイルに永続化される前に、変数インジェクタまたはモデル・フィルタがその順序で実行されます。 最後のステップはバリデーションで、モデル、アーカイブおよび変数ファイルの内容をバリデートします。 バリデーションが成功した場合、モデルは保持されます。 これらの3つのトピックの詳細は、次を参照してください:

  • 可変インジェクション
  • モデル・フィルタ
  • Validate Model Tool

結果のモデルは、Oracle WebLogic Server Kubernetes Operatorなどの特定のターゲット環境との互換性のために変更することもできます。 詳細については、「ターゲット環境」を参照してください。

問題(または成功)は、Discover Domain Tool サマリーは、ツールのバージョンとOracleホーム、およびツールが実行されたWLSTモード(オンラインまたはオフライン)を出力します。 すべての警告および重大メッセージの要約が、合計とともにリストされます。

警告メッセージを含むサマリーの例:

Issue Log for discoverDomain version 1.5.2-SNAPSHOT running WebLogic version 10.3.6.0 offline mode:

WARNING Messages:

        1. WLSDPLY-06200: Unable to get the Security Realm Provider name in version 10.3.6.0 with offline wlst. The SecurityConfiguration will not be added to the model. The work-around is to manually add the Security Configuration to the model or to discover the domain in online mode : Invalid Security Provider name "Provider" found for provider type at location /SecurityConfiguration/Realm/Adjudicator.

Total:       WARNING :     1    SEVERE :     0

環境変数

次の環境変数を設定できます。

  • JAVA_HOME JDKのロケーション。 これは有効なJava 7以降のJDKである必要があります。
  • WLSDEPLOY_HOME WebLogic Deploy Toolingインストールのロケーション。 デフォルトでは、ロケーションはdiscoverDomainスクリプトのロケーションから計算されます。
  • WLSDEPLOY_PROPERTIES WLSTに渡されるシステム・プロパティ。

Discover Domainに対する問題のオープン

GitHub問題にSTDOUTおよびSTDERRログ・ストリームを指定してください。 サマリーがリストに表示されない場合(未処理の例外スタック・トレースが発生した場合)、OracleおよびWDTのインストール・バージョン、およびツールがオンラインまたはオフラインのWLSTモードで実行されたかどうかを確認してください。 可能な場合は、<install home>\weblogic-deploy\logからモデル、変数およびアーカイブ・ファイルおよびログ・ファイルdiscoverDomain.logを指定します。

discoverDomainのパラメータ表

パラメータ 定義 デフォルト
-archive_file アーカイブ・ファイルへのパス。
-admin_pass_env プロンプトで管理パスワードを入力する代替手段。 値は、WDTがパスワードの取得に使用する環境変数名です。
-admin_pass_file プロンプトで管理パスワードを入力する代替手段。 値は、ツールがパスワードを取得するために読み取るパスワード文字列を含むファイルの名前です。
-admin_url オンライン検出に使用される管理サーバーURL。
-admin_user オンライン検出に使用される管理ユーザー。
-domain_home (必須) 既存のドメイン・ホームのロケーション。
-domain_type ドメインのタイプ(WLS、JRFなど) WLS
-java_home Javaホーム・グローバル・トークンに置換されるドメイン値を検出するときに、JAVA_HOME値をオーバーライドします。
-model_file モデル・ファイルへのパス。 存在しない場合、モデル・ファイルはアーカイブ・ファイルに格納されます。
-oracle_home Oracle WebLogicインストールのホーム・ディレクトリ。 ORACLE_HOME環境変数が設定されていない場合は必須です。
-output_dir -targetには出力ディレクトリが必要です。
-skip_archive アーカイブ・ファイルは生成しないでください。 -archive_fileオプションは無視されます。
-target ターゲット出力タイプ。 デフォルトはwkoです。 ターゲット・タイプの詳細は、「ターゲット環境」を参照してください。
-remote リモート・マシンからドメインを更新します。
-variable_file ♪変数プロパティ・ファイルへのパス。
