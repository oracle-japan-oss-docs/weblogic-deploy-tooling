リリース3.0.0

リリース3.0.0での変更点

  • 主な新機能
  • その他の変更
  • バグ修正
  • 既知の問題

主な新機能

  • #1355: リモート・マシンからのアプリケーションおよび共有ライブラリのデプロイをサポートする-remoteオプションをdeployAppsツールに追加しました。 (問題#1312)
  • #1355: -remoteオプションがupdateDomainツールに追加されました。このツールでは、リモート・マシンからアーカイブ・ファイルの変更を行わなくても(アプリケーションと共有ライブラリを除く)、構成を変更できます。 (問題#1167)
  • #1365: モデルの新しいverrazzanoセクションが追加され、Verrazzano関連の-targetオプションによって生成されたYAMLファイルを拡張/オーバーライドできます。 (WDT-688)
  • #1367: コマンドラインからアーカイブ・ファイルを作成および更新するための新しいarchiveHelperツール。 (WDT-711)
  • 別名をクリーン・アップしてオンライン検出をクリーン・アップし、様々なWebLogic ServerリリースおよびPSUと適切に連携するようにテストを配置しました。 オンライン検出の結果のモデルは、オフライン検出によって生成されたモデルと大幅に近くなります。

その他の変更

  • #1255 : オフライン・ネーミング計画に合せてモデル属性の名前を変更しました:

    • SystemComponent MWHomeモデル属性の名前がMwHomeに変更されました。
    • WebserviceSecurity DefaultCredentialProviderSTSURIモデル属性の名前がDefaultCredentialProviderStsuriに変更されました。

  • #1299 : オフライン・ネーミング計画に合せてモデル属性の名前を変更しました:

    • SecurityConfiguration IdentityAssertionCacheTTLモデル属性の名前がIdentityAssertionCacheTtlに変更されました。

  • #1310: 複数の12.2.1.3 PSUのPSUバージョンが正しくない原因となった解決済みの問題。

  • #1314, #1342: production modeおよびsecure production modeの使用時にデフォルト値がどのように変化するかを理解するための知識が別名に追加されました。 (WDT-678)

  • #1328: 警告が存在する場合は終了コード1で終了し、サマリー・レポートにエラーが存在する場合は終了コード2を終了するようにツール終了コード処理を変更しました。 (WDT-692)

  • #1331: アーカイブ・ファイル内のOPSSウォレットおよびATPウォレットの古いアーカイブのロケーションが非推奨になりました。 古いロケーションは有効ですが、1つ以上の非推奨メッセージになります。 (WDT-687)

  • #1334: モデル・ファイルをアーカイブに格納するためのサポートが削除されました。 (WDT-643)

  • #1341: extractDomainResourceツールから非推奨の-domain_resource_file引数を削除します。 (WDT-706)

  • #1348: 古いATPウォレットの名前をrcuウォレットに変更しました。 (WDT-709)

  • #1348: ZIPファイルまたは1つ以上のファイルを含む展開されたディレクトリとしてウォレットを追加するためのサポートが追加されました。 ウォレットがZIPファイルとして追加されると、createDomainおよびupdateDomainツールは、以前と同様にZIPをドメイン・ホームに展開します。

  • #1348: 複数のウォレットに対するデータベース・ウォレットのサポートが追加されました。 必要に応じて、他のデータベース・ウォレットを追加できます。 (WDT-710)

  • #1351: モデルのkubernetesセクションから非推奨の「名前付きオブジェクト・リスト」形式のサポートが削除されました。 (WDT-688)

  • #1366: ATPとSSLのデータベース・サポートがリファクタリングされ、モデリング・アプローチで統合されます。

  • #1369: ゼロ以外の終了ステータスを発生させずにツールのサマリー・レポートに表示されるように、非推奨メッセージの特別なログ・レベルが追加されました。 (WDT-721)

  • #1375: アプリケーション・インストール・ディレクトリ全体を取得するために、アプリケーション・インストール・ディレクトリの検出が改良されました。 (WDT-715)

  • #1379: アーカイブ内のJDBCウォレットfile(s)を収集する検出機能が追加されました。

  • #1380: イングレス・トレイト・ルーティング・ルールに関連するVerrazzanoカスタム・リソースを生成するアプローチを更新して、宛先ホスト/ポートを追加し、ユーザーがモデルの新しいverrazzanoセクションを介してパスを追加できるようにします。 (WDT-696)

  • #1383: 検出されたモデルからDomainVersionドメイン・レベル属性を削除し、モデルを検出するために使用されたWDTおよびWebLogic Serverバージョン番号と、検出に使用されたWLSTモードについて説明するコメントに置き換えました。 (WDT-698)

  • #1386: 変更された属性は、かわりにINFOレベルのメッセージをログに記録するために、現在のWebLogicバージョンの警告では有効ではありません。 (WDT-724)

  • #1389: ドメイン作成前にモデリングの問題を捕捉するために、動的クラスタに関する追加のモデルおよびWebLogic Kubernetes Operator/Verrazzanoターゲット・バリデーションが追加されました。 (WDT-726)

  • #1391: リモート検出を使用するときに生成されたツール・サマリー・レポートおよびリモート・レポート。 (WDT-637)

  • #1395: クラスタ化されていないサーバーをprepareModel (WDT-730)のWKTUI統合出力に追加しました

バグの修正

  • #1261, #1266: キーストアが見つからないときにdiscoverDomainエラー・メッセージを修正しました。 (WDT-685)
  • #1324: セキュア・モードが検出されない原因となる問題が解決されました。 (WDT-694)
  • #1327: WLS 12.2.1.0.0のWLSTエラーがツールのサマリー・レポートに表示されているという問題を解決しました。 (WDT-691)
  • #1329: ツール・サマリー・ログ・ハンドラを変更して、WDTに適切に生成されなかったメッセージを受け入れないようにしました。 (WDT-701)
  • #1330: Windowsでの環境変数置換の問題を修正しました。この場合、モデル内で大文字ではない環境変数参照が正しく動作していませんでした。
  • #1335: deployAppsおよびupdateDomainツールから、未使用およびドキュメント化されていない-prev_model_file引数が削除されました。 (WDT-707)
  • #1344: モデルをアーカイブ・ファイルに格納できなくなったため、extractDomainResourceツールから-archive_file引数を削除しました。 (WDT-708)
  • #1367: 同じ名前の既存のディレクトリがすでに存在する場合に、ディレクトリの名前を変更するようにアーカイブ・クラスの動作を修正しました。 これにより、ファイルおよびディレクトリの動作に一貫性があります。 (WDT-722)
  • #1374: カスタム構成ファイルを持つCoherenceクラスタを使用したドメインの検出に関する問題を修正しました。
  • #1376: JVM引数と空白の比較に関するcompareModelの問題が修正されました(問題#1370)
  • #1382: 起動時に、PSU情報なしでWebLogic Serverバージョンを表示した誤解を招くログ・エントリが削除されました(WDT-727)
  • #1392: 動的サーバーが誤って構成済サーバーとして追加された動的クラスタを含むドメインのオンライン検出の問題が修正されました。
  • #1393: オンラインで検出されたモデルに空のSecureModeフォルダが表示される原因となる問題を修正しました。 (WDT-731)
  • #1394: オンライン検出中に別名エントリに関連するエラーのあるINFOメッセージの一部を回避するための別名エントリが修正されました。
