サンプル

これらの一般的なユースケースのシナリオでは、モデルで共通構成がどのように表されるかを示します。

    管理サーバーのカスタマイズ

    ドメイン・モデルを使用して管理サーバーを構成します。

    構成済クラスタのモデリング

    構成済クラスタの一般的な構成を持つドメイン・モデル。

    WTC構成のモデリング

    WebLogic Tuxedoコネクタの一般的な構成を持つドメイン・モデル。

    JDBCデータ・ソースのモデリング

    JDBCデータ・ソースの一般的な構成を持つドメイン・モデル。

    ワーク・マネージャのモデリング

    ワーク・マネージャの一般的な構成を持つドメイン・モデル。

    セキュリティ・プロバイダのモデリング

    WebLogic Serverセキュリティ構成の特別な処理およびモデル・セマンティクス。

    Oracle Platform Security Servicesのモデリング

    WebLogicドメイン作成の一部としてOPSS構成を初期化しています。

    WebLogicユーザー、グループ、ロールおよびポリシーのモデル化

    WebLogicドメインの作成の一環として、ユーザー、グループ、グローバル・ロールおよびポリシーを確立します。

    WebLogicユーザー・パスワード資格証明マッピングのモデリング

    ユーザー・パスワード資格証明マッピングを作成するためのモデル。

    Oracle Diagnostic Loggingの構成

    Oracle Diagnostic Logging (ODL)を構成するためのモデル。

    Oracle HTTP Serverの構成

    Oracle HTTP Server (OHS)を構成するためのモデル。

    サーバー・グループのターゲット指定

    管理対象サーバーおよび動的サーバー・グループへのサーバー・グループの動的クラスタへのターゲット指定。

    WebLogic Kubernetes OperatorでのWDTの使用

    WebLogic Kubernetes OperatorでWDTを使用します。
