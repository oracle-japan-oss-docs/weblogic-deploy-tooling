WebLogic Deploy Tooling

WebLogic Deploy Tooling (WDT)を使用すると、WebLogic環境を簡単に立ち上げ、プロジェクトの進化に応じて処理および展開できるメタデータ・モデルに基づいて、繰り返し可能な方法でドメイン・ライフサイクル操作を実行できます。

多くの組織は、エンタープライズ・アプリケーションを実行するために、他のOracle Fusion Middlewareコンポーネントとの間でWebLogic Serverを使用します。 さらに、これらの組織がアプリケーションの継続的デリバリに移行するにつれて、自動構成アプリケーションのデプロイメントの重要性が増します。 この自動化は、多くの場合、WebLogic Scripting Tool (WLST)構成およびデプロイメント・スクリプト言語を使用して実装されますが、これは困難です。 プロジェクトが展開したり、テストや本番など、プロジェクトが異なる環境にデプロイされるため、このようなスクリプトは慎重に更新する必要があります。

WebLogic Deploy Toolingでは、ほとんどのWebLogic Serverデプロイメントで、ルーチン・ドメイン作成タスクおよびアプリケーション・デプロイメント・タスクを自動化するために、手動コード化されたWLSTスクリプトに依存する必要がなくなります。 宣言的なメタデータ・モデルを作成して、ドメインとアプリケーション(依存リソースを含む)を記述し、モデルのコンテンツに基づいてドメイン・ライフサイクル操作を実行するための単一目的のツールを提供します。 また、テストと本番の間など、様々な環境間でモデルの移動に適した単純な突然変異を指定することもできます。

WDTには、シェル・スクリプト(WindowsとUNIXの両方)として公開される複数の単一目的ツールがあり、次のことが可能です:

  • ドメインを作成または更新します。
  • モデルで指定されたすべてのリソースおよびアプリケーションをドメインに配置します。
  • 既存のドメインにリソースおよびアプリケーションを追加します。
  • 既存のドメインをイントロスペクトし、ドメインとドメインにデプロイされたバイナリのアーカイブ・ファイルを記述したモデル・ファイルを作成します。
  • モデル(またはその変数ファイル)のパスワードを暗号化します。
  • モデルをバリデートし、モデル使用方法情報を提供します。
  • モデル・ファイルを比較します。
  • WebLogic Kubernetes Operator環境にデプロイするためのモデル・ファイルを準備します。
  • WebLogic Kubernetes Operatorで使用するドメイン・リソースYAMLファイルを生成します。
  • 変数を使用してモデルをトークン化します。
  • ドメイン・モデルのセクションおよびフォルダに対して有効なフォルダおよび属性に関する情報を指定します。

詳細については、「WDTツール」を参照してください。

このドキュメントについて

このドキュメントには、さまざまなオーディエンスを対象としたセクションが含まれています:

  • 「概念」では、基礎となるメタデータ・モデルおよびアーカイブ・ファイルについて説明します。
  • 「ユーザー・ガイド」には、WebLogic Deploy Toolingのインストールおよび構成方法、各ツールの使用方法など、詳細な使用状況情報が含まれています。
  • 「サンプル」は、有益なユースケース・シナリオを提供します。
  • 「開発者ガイド」には、WDTの構築方法、その機能のマップおよび実装方法を理解するユーザーの詳細が表示されます。 WebLogic Deploy Toolingコードに関与したい方は、「こちら」という有用な情報を検索します。
